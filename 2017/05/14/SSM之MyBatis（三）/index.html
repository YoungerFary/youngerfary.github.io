讲解下这是什么意思，首先我们得到的是一个resultMap的结果集，然后再结果集中我们注意的是其链接的是teacher对象，关键字用的是association，property是teacher，后面是类型javaType，这里最主要的就是这个association！

方式二：嵌套查询：通过执行另外一个SQL映射语句来返回预期的复杂类型
SELECT FROM class WHERE c_id=1;
SELECT FROM teacher WHERE t_id=1 //1 是上一个查询得到的teacher_id的值
这里我们可以知道，分成了两步，第一步找到了class,第二步根据id去找teacher的信息，事先没有链接！

这里我们仔细看resultMap中最后的association标签后面的，column得到的是teacher的id,select表示他要去哪查询！

5、测试

这里我们对关键字做个总结：
1、assosiation:用于一对一的关联查询
2、property:对象属性的名称
3、javaType:对象属性的类型
4、column:所对应的外间字段名称
5、select:使用另一个查询分装的结果

根据classId查询对应的班级信息,包括学生,老师，我们知道一个班级中有多个学生，一个老师对应一个班级！这是他们之间的关系！

4、定义sql映射文件ClassMapper.xml

这个也有两种方式，道理都是和上面一样的

(1)方式一: 嵌套结果: 使用嵌套结果映射来处理重复的联合结果的子集

SELECT * FROM class c, teacher t,student s WHERE c.teacher_id=t.t_id AND c.C_id=s.class_id AND c.c_id=1

如上所示，老师和班级是一对一的关系，而班级和学生是一对多的关系，所以我们用了collection关键字，这是个集合，ofType指定students集合中的对象类型，
后面是一样的映射！

（2）方式二：嵌套查询：通过执行另外一个SQL映射语句来返回预期的复杂类型

SELECT * FROM class WHERE c_id=1;
SELECT * FROM teacher WHERE t_id=1   //1 是上一个查询得到的teacher_id的值
SELECT * FROM student WHERE class_id=1  //1是第一个查询得到的c_id字段的值

如上所示定义的是不同的select,传过去的column是teacher_id和c_id，其余都差不多！

5、测试

关键字解释：
1、collection:做一对多关联查询的
2、ofType:指定集合中元素对象的类型！
